## ðŸŽ Bonus: Common Follow-Up Questions

**Q: "What if you have limited time for regression?"**
Answer: "I'd run critical smoke tests first (15-30 min), then high-priority sanity tests (1 hour), and defer exploratory testing. I'd communicate the risk clearly to the team."

**Q: "How do you handle flaky tests?"**
Answer: "I investigate root cause (timing, data, environment), refactor test to be more reliable, and consider if it should remain automated or move to manual."

**Q: "What metrics do you track?"**
Answer: "Test pass rate (target >95%), regression cycle time (target <2 hours), code coverage (target >80%), and bug escape rate (target <1%)."

**Q: "How do you work with Dev during testing?"**
Answer: "Collaboratively - I report bugs with clear reproduction steps, discuss alternatives, and retest fixes. I view quality as shared responsibility."
